系統設計核心

本系統旨在輔助高中學生整理課程學習成果、多元表現、自主學習計畫、升學自傳與讀書計畫。系統遵循大學招生審查之「三重二不」精神（重素養、重校內、重真實；不以數量取勝、不要求樣樣具備），以「學生真實經歷」為本，提供本機診斷及 AI 修飾建議。

系統理念：AI 是教練與鷹架，不是代筆。工具用於引導發想、修辭建議、結構重建與模擬追問；核心的核心仍必須是學生自己的真實反思與實際經歷，如此才能在備審資料與面試時維持誠信與說服力。

核心運作模式

01

本機快速診斷

無須連網及 API Key，完全在學生本機端以關鍵字特徵與規則進行即時評分與缺口診斷，保護隱私。

02

二階段深度分析

透過 AI 對照「完整性、獨特性、關聯性、真實性、策展主軸、第二階段準備」進行六大面向之核心評估與策略引導。

03

AI 提示詞生成

為了解決直接叫 AI 寫文章容易流於罐頭氣息的缺點，系統會將學生的背景、申請校系、凸顯能力、佐證素材與原始草稿，編織成高客製化、避免虛構的「高階修飾提示詞」，讓學生貼往 ChatGPT 或 Gemini 取得精準成果。

04

API 連線直呼

輸入本機 API Key 後，可直接在網頁呼叫 OpenAI 或 Google Gemini API 快速生成回覆，加速迭代。

學生檔案管理 (頂部工具列)

此區塊位於網頁最上方，負責記錄多位學生的填寫狀況，將資料加密儲存於瀏覽器本機快取 (LocalStorage)，不儲存於伺服器，安全無個資外洩風險。

按鈕或欄位名稱 功能操作說明 學生姓名 (輸入框) 在儲存前先在此欄位填入學生姓名或作品代號。本姓名將作為本機資料庫儲存與匯出 Word 檔時的檔名。 選擇已儲存學生 (下拉選單) 列出目前在瀏覽器中已經儲存過的所有學生紀錄。點選後可切換載入不同學生的資料。 讀取 在下拉選單選擇好學生後，點選此按鈕載入對應之背景設定、草稿內容與過往診斷紀錄。 儲存 將目前畫面上所有設定與填寫內容，儲存至瀏覽器本地資料庫中。建議在修改各分頁資訊後，隨時點選此按鈕以防資料遺失。 刪除 刪除目前在下拉選單選取學生的本機備份紀錄。動作執行前會有確定對話框提示。 下載 Word 檔 一鍵將目前學生的申請背景、原始草稿、本機快速診斷報告、AI提示詞及 AI 產出回覆，整合成一個排版美觀的 .doc 格式 Word 檔案，讓學生能印出或在單機進行深度修改與備份。 AI 設定 點選後開啟浮動視窗，設定 OpenAI 或 Google Gemini API 的連線金鑰與模型選擇（詳見第 6 節說明）。

分頁一：申請與素材設定

此頁面是系統的起點。提供越具體、清晰的申請目標與佐證素材，系統產生的本機評估與 AI 提示詞便會越精準、越貼合校系需求。

設定欄位詳細操作說明

設定項目 欄位類型 填寫與操作指南（降低難度指引） 檔案類型 下拉選擇 選擇本次欲優化的備審資料種類。包含：
  • 課程學習成果 (如探究報告、課堂簡報)
  • 多元表現 (如幹部心得、競賽紀錄)
  • 升學自傳、學習歷程自述
  • 自主學習計畫與成果
  • 探究與實作成果、讀書計畫
  • 多元表現綜整心得 等。
檔案名稱 文字輸入 填入這份成果或計畫的標題。
範例：水質檢測專題成果報告、麗山高中校慶攤位規劃心得。 申請大學 文字輸入 填入目標申請的大學校名。此資訊會用於 AI 連結分析。
範例：國立陽明交通大學、國立臺灣大學。 申請科系 文字輸入 填入目標科系。系統診斷會自動對比科系核心能力。
範例：生物科技學系、資訊工程學系。 想凸顯的能力 多選核取方塊 + 自訂文字區 請學生勾選此份檔案中最想讓教授看見的核心素養與特質（如自主學習、問題解決、反思修正等）。
若有其他非勾選區特質，可在下方「其他能力補充」文字框中以逗號隔開輸入。 可佐證素材 多選核取方塊 + 自訂文字區 勾選在最終成果 PDF 檔案中已經包含的佐證材料（如數據表、實驗照片、教師回饋、修正前後對比等）。這能幫助本機診斷及 AI 評估證據的說服力強度。

分頁二：學習歷稿工作區與診斷

這是最核心的修改與互動空間。學生在此輸入原始草稿，並透過本機的多種演算法及 AI 提示詞，進行全方位的診斷與潤飾。

1. 上傳檔案解析與圖文檢核區

學生可以直接上傳 PDF 檔或 Word .docx 檔案。系統會嘗試抽取純文字至草稿區。

  • 讀取檔案到草稿區：將上傳檔案的文字解析並自動填入下方原始草稿區。
  • 開啟 PDF 原檔檢視：開啟網頁內嵌式檢視器，讓學生可以一邊修改草稿、一邊對照原本 PDF 的排版與圖表。
  • PDF 圖文檢核：對上傳 PDF 進行結構、圖表數量、表格、圖片密集度檢測，並在下方的「診斷」頁籤生成對照提醒。

2. 診斷與優化控制按鈕操作說明

診斷按鈕 適用對象與功能說明 點擊後的處理流程與預期結果 本機快速診斷 初稿檢測。 快速掃描字數、段落分配、反思語彙（困難/修正）、佐證字彙與科系字彙。 結果會呈現在下方 診斷 頁籤，包含結構、佐證、反思、科系連結、真實性五個面向的 0-100 分量化圖表與修正優缺點清單。 第二階段深度分析 送件前把關。 依據臺灣第二階段審查邏輯進行六大深度檢核。 提供優先補強順序，並在本機產生 8 題面試口試教授極可能追問的「備審真實性」問題。 面試準備診斷 口試前準備。 評估經歷是否能轉化成口語說法。 分析是否具備一分鐘自我介紹、START敘事架構與壓力題應變策略，並生成 10 題高階面試追問題。 讀書計畫診斷 讀書計畫專用。 檢查未來升學規劃。 檢測是否流於「努力讀書」等罐頭詞彙、是否客製化、是否具備可檢核指標，並列出地雷警示。 自傳優化診斷 自傳專用。 評估自傳的品牌契合度。 檢查自傳是否有黃金開場、是否避開家庭背景流水帳，並提供優化架構藍圖。 產生 AI 修飾提示詞 無 API Key 或希望在外部發問者。 根據目前設定，在下方 AI提示詞 頁籤產生極詳細的高階 Prompt。可直接複製後點選下方 開啟 ChatGPT 或 開啟 Gemini 貼上。 用 API 產生回覆 已填寫 API Key 的使用者。 直接呼叫 API 獲取修潤、重建段落、產生摘要等成果，並在下方 AI回覆 頁籤中呈現。 清空 重置整個工作區。 清空所有暫存輸入與檔案（已手動儲存的學生檔不會被刪除）。

3. 下方結果頁籤 (Tabs) 操作說明

點選各頁籤可查看對應按鈕所產生的不同產物：

  • 診斷：查看本機評分、警告訊息、優點與下一步修改指引。
  • 摘要：顯示產生的精簡摘要（如為誠信聲明，點選「產生誠信聲明」按鈕後，會在此頁籤顯示 AI 使用聲明草稿）。
  • AI提示詞：在此切換欲向 AI 詢問的任務任務（如：修飾與優化草稿、模擬審查委員回饋、產生面試追問題等），再點選下方複製按鈕或一鍵開啟外部 AI 平台。
  • AI回覆：直連 API 回覆的內容，附有複製按鈕，方便將修飾後的文本帶回 Word 修改。

分頁三：學生參考資料與列印

此頁面整合了高中學習歷程檔案審查的各項寫作規範與檢核，建議提供給學生作為修改時的隨手核對清單。

  • 一鍵列印設計：點選 列印本頁參考資料 按鈕，系統會自動調整為適合 A4 列印的黑白排版版面，方便老師影印給班級學生或學生線下閱讀。
  • 資料卡片庫：包含整體審查精神、課程學習成果寫作結構、多元表現寫作重點、自傳 STAR 經歷寫作法、讀書計畫客製化防雷、面試自我介紹策略等。

AI 連接設定 (API 設定)

點選頂部工具列的 AI 設定 可設定 API 連線金鑰，本金鑰僅保存在使用者的瀏覽器，不回傳至伺服器。

安全提示：若是使用學校電腦教室等公用電腦，請在「API 設定儲存方式」中務必選擇「暫時儲存，關閉瀏覽器後清除」，以免個人 API Key 遭到後續使用者挪用。

設定選項解說

  • AI 服務：選擇 OpenAI Responses API 或 Google Gemini API。
  • 模型：預設提供快速且省額度的模型（如 gpt-5.4-mini、gemini-3.5-flash）。高需求或遇到限額時，可按「改用快速模型」切換至備援。
  • 溫度 (Temperature)：數值越低（如 0.2~0.4）生成的內容越嚴謹、符合事實；數值越高（如 0.7~0.9）生成內容越有創意。本系統預設為 0.4，在修飾學生經歷時能維持高度真實性，避免 AI 胡言亂語。
  • API Key 申請：浮動視窗中提供官方直連網址，點選即可跳轉至 OpenAI Platform 或 Google AI Studio 進行申請與付款管理。

誠信使用指引與誠信聲明

為避免學生過度依賴 AI 產生虛構經歷，或因使用了大量 AI 詞彙而使學習成果顯得虛假，請務必遵循以下使用指引：

1. 移除個資：在送交 AI 診斷或修飾前，請確認草稿中已移除身分證字號、姓名、家長姓名、手機號碼與詳細住址等敏感個資，確保隱私安全。
2. 內容查證：AI 生成的讀書計畫若提及該科系的特定課程、教授、研究中心或合作實習單位，學生必須親自前往該大學官網進行「二次查證」，確認該資源目前仍存在且適合自己。
3. 二次改寫與熟練：AI 修改後的任何優美段落或面試追問題，學生都必須能用自己的口吻自然地講述出來。若面試時教授追問細節而學生答不上來，將會被判定為代筆，產生極大升學風險。
