系統簡介與特色

本系統是專為高中自主學習（彈性學習時間）設計的引導網頁。許多學生在規劃自主學習計畫時，常面臨「不知道要學什麼」、「動機太過空泛」、「目標無法具體檢核」等困境。本工具旨在以結構化的五步流程引導學生填寫，並配合 AI 輔助生成技術，化身為專屬學生的自主學習計畫起草教練。

💡 教師免責好處： 本程式完全運作於瀏覽器本機，不需註冊帳號，所有資料皆儲存在使用者本機瀏覽器中（localStorage）。學生換電腦或清除瀏覽器快取時，資料才會消失。這意味著學校不需架設伺服器，也無個資外洩的隱憂。

五大核心特色：

  • 分步引導設計：將繁瑣的官方計畫書拆解成五個簡潔的填寫分頁，循序漸進。
  • 內建引導語與範例：每個欄位皆附帶「引導思考角度」與「具體範例」，學生看著就能下筆。
  • 雙重 AI 模式：「外部模式」保護個資且完全免費；「即時模式」提供一鍵式生成體驗。
  • 自動化日期與檢核：輸入起始日期，自動代入週次日期；內建 A/B/C 自我檢核追蹤系統。
  • 多元化匯出：支援一鍵複製、下載微軟 Word 檔、或以 A4 紙本規格預覽列印（可另存 PDF）。

頂部學生存檔功能

位於網頁最上方的「學生資料列」是本機端的資料暫存中心。由於沒有資料庫，本功能使用瀏覽器的 localStorage 來存取多個學生的進度，非常適合同一台電腦由多位同學共用，或同一位同學多次修改的情況。

按鈕/欄位 功能說明 操作提醒
學生下拉選單 顯示這台電腦上所有已儲存的學生姓名（以 slp_[學生姓名] 為 LocalStorage 的 key 保存）。 選擇後必須按「讀取」才會將資料載入表單。
讀取 將下拉選單中選取學生的計畫書資料，完整載入到當前網頁中。 讀取會覆蓋目前畫面上已輸入的暫存內容，讀取前請確認目前內容已儲存。
儲存 以「申請人姓名」為關鍵字，將畫面上所有欄位、週計畫表、學習成效紀錄與已上傳的圖片儲存至本機。 重要：若未填寫第一頁的「申請人姓名」，將無法儲存。
刪除 從本機瀏覽器中徹底刪除選定學生的資料。 此操作不可逆，請小心點選。系統會跳出二次確認視窗。
清空表單 清除目前網頁畫面的所有填寫內容，還原為全新的空白表單。 不會影響到已經按「儲存」保存在本機的學生資料。

⚙️ AI 輔助核心設定

點擊可展開「⚙️ AI 輔助設定」區塊。系統設計了極具彈性的 AI 輔助模式，降低老師與學生的使用障礙與費用壓力：

🤖 外部模式 vs 即時模式
  • 外部模式（預設 - 免費免設定）：當「API 金鑰」欄位留空時，即啟用此模式。當學生點擊任何欄位旁的「AI 協助」時，系統會自動在剪貼簿中產生量身打造的 Prompt（指令），並自動在新分頁開啟指定的 AI 網站。學生只需按 Ctrl+V（Mac 用 Cmd+V）貼上，即可在官方網站中獲得解答。
  • 即時模式（免跳轉 - 需填寫 API Key）：若學生或老師填入 API 金鑰，則會在網頁內直接呼叫 API。生成內容會呈現在欄位下方，並提供「套用到欄位」按鈕，實現在同一網頁內一鍵優化。

設定欄位與操作按鈕：

  • 使用的 AI 服務：可選擇 ChatGPT、Claude 或 Gemini。外部模式下會開啟對應的對話連結；即時模式下會發送 API 到對應平台。
    • 註：ChatGPT 選項已客製化 GPT Link，會直接導向特化版本的專用 GPT。
  • 即時模式 API 金鑰：填入 OpenAI API 金鑰 (sk-...)、Anthropic 金鑰或 Gemini 金鑰。金鑰只會保存在本機瀏覽器，無洩漏風險。
  • 模型名稱：若留空，系統會以當前性價比最高的預設模型呼叫（ChatGPT: gpt-4o-mini，Claude: claude-3-5-sonnet-latest，Gemini: gemini-2.5-flash）。
  • 儲存設定：將金鑰與服務商設定儲存在本機。儲存成功後，狀態標籤會由灰色的「外部模式」變為綠色的「即時模式」。
  • 清除金鑰：一鍵清除瀏覽器暫存的金鑰，立刻切回安全的外部模式。

1. 基本資料頁

此頁主要對應自主學習計畫書的封面資訊。必須在起草階段優先完成，以利後續 AI 順利讀取背景脈絡。

欄位與功能操作：

  • 申請人姓名（必填）：輸入學生姓名。此姓名也是本機「儲存功能」的主索引，請務必填寫。
  • 班級、座號：輸入班級座號（例如：高二3班、15號），將自動帶入最終計畫書。
  • 計畫主題（必填）：用一句話描述自主學習大方向。網頁附有引導提示與範例。
  • 計畫類型（必填）：點選單選按鈕。
    • 當點選「7 專題研討」時，系統會自動連動展開「專題研討領域」的複選方塊（國文、英文、數學、物理、化學、生物、地科、資訊等），學生可依學科多選。
  • 實施方式（必填）：個人學習、小組學習。
    • 當點選「小組學習」時，系統會自動連動展開「小組成員」輸入框，供填寫同組組員姓名（上限 3 人）。
  • 計畫名稱（必填）：更具體的正式計畫名稱（例如主題是「固碳能力」，計畫名稱可為「校園喬木與灌木固碳量之比較與綠化建議」）。
  • 詳細主題 / 簡短說明：補充計畫的探究範圍、遇到的痛點或好奇的細節。

本頁按鈕：

🤖 AI 協助：把主題想得更清楚

AI 後台運作邏輯（Prompt）：讀取上述所有欄位，幫學生評估此主題是否太大或太小、給出 2-3 個更聚焦的建議題目、指出可探究的核心問題，並提醒可行性與資源上的注意事項。

2. 學習計畫頁（內容概述）

本分頁為整份計畫書的「靈魂核心」，對應官方格式中的動機、資源、方法與目標四大部分。

💡 指導策略： 老師可要求學生先在欄位中寫下「一句話的模糊想法」，接著點選對應的 AI 協助按鈕，讓 AI 引導深化，學生再將其修改後貼回。這種「人機協作」能大幅減少學生寫不出長文的挫折感。

四大核心欄位與引導：

  1. 學習動機（為什麼想做這個主題？）：引導思考好奇的點、與未來升學或生活的關聯性。
  2. 我需要什麼資源或工具？：引導思考書籍、網站、論文、軟體、場地與費用。
  3. 我如何有效學習？（方法或技巧）：引導具體學習步驟、策略，以及自我檢核機制。
  4. 預期目標：引導撰寫具體、可衡量的期末產出（例如完成一份 1500 字的實驗報告與一張圖表）。

本頁按鈕功能：

按鈕 功能與 AI 提示工程邏輯
🤖 AI 協助這欄
（各欄位旁）		 針對單一欄位進行補強。例如，「動機輔助」會將學生簡短的文字修潤成有個人連結且具說服力的條列版；"目標輔助"會將目標具體化，並給予「基本目標」與「進階目標」的階梯式版本。
✨ AI 統整全計畫 整體檢視按鈕。會同時讀取動機、資源、方法、目標四個欄位，分析彼此是否具備邏輯一致性（例如：方法有沒有用到資源？目標有沒有呼應動機？），並提供一份潤稿好的「內容概述」整合建議。
🔎 上網查資料 自動以學生輸入的「計畫名稱」或「計畫主題」作為關鍵字，開新分頁進行 Google 搜尋，方便找論文或線上課程資源。

3. 預計進度頁（週計畫）

週計畫要求學生將計畫拆解到每週實施。本頁提供便利的自動化表格生成，大幅省去學生手動排版、填寫週次的麻煩。

操作步驟：

1

設定週數與起始日期

在下拉選單選擇「總週數」（1-20 週），並選擇「起始日期」。系統會自動以每 7 天為一週計算，自動算出每週對應的日期。

2

產生進度表

點選 產生 / 更新進度表，系統會即時在下方渲染出一個包含「週次、日期、進度內容、自我反思檢核（A/B/C 下拉選單）、老師簽名欄」的表格。

3

AI 進度規劃（選填）

如果學生不知道每週該做什麼，可點選 🤖 AI 設計每週進度與檢核。AI 會根據第二頁的學習方法與目標，規劃符合週數的「每週進度 | 檢核方式」大綱，學生可以直接參考並填入表格中。

💡 設計巧思（Milestone）： 生成的進度表中，最後倒數兩週的進度內容會自動顯示預設的 Placeholder 建議（「自主學習成果分享」與「自主學習成果反思」），且該兩列會以淡藍色背景高亮顯示，以符合教育部倡導「最後階段應有分享與反思」的計畫書規範。

4. 學習成效頁

此分頁是「執行自主學習計畫期間與期末」才需要填寫的。它用於追蹤記錄實際執行狀況，也是期末成果報告的底稿。

主要功能與按鈕：

  • 依進度週數產生成效表：點擊後，系統會依據前一頁設定的週數，在下方渲染出「週次、學習成效、成果影像」表格。
    • 學習成效（文字）：學生可在執行當週或期末，逐週輸入該週實際學到、做到什麼，以及遇到的問題與解決方式。
    • 成果影像（多媒體）：提供 📷 上傳 按鈕。學生可直接將當週的成果照、筆記截圖、實驗照片上傳。系統會自動將圖片轉為 Base64 編碼，直接嵌入並預覽（亦可一併保存在 LocalStorage 中）。若想刪除，可按紅色的 🗑 垃圾桶按鈕。
  • 🤖 AI 協助潤飾成效紀錄：讀取學生目前在成效表格中輸入的所有內容，由 AI 協助進行文字重組與優化，使其文筆更具學術性與反思深度。
  • 自我檢核次數統計：系統會自動掃描「預計進度頁」中每一週的下拉選單（A/B/C），統計出次數。若學生在中途有更新進度頁的下拉選單，可點選下方的 重新計算統計 按鈕來更新次數。
    • A 依計畫完成 / B 修正後完成 / C 部分完成。
  • 家長回饋與簽名：供家長在檢視學生計畫與成果後填寫評語，並鍵入姓名。
  • 成果發表形式（複選）：勾選期末的發表方式。若勾選「其他方式」，可於下方的輸入框中手動輸入細節。

5. 匯出列印頁

當基本資料、內容概述、週進度與成效紀錄皆填寫完畢後，本分頁提供一鍵產出正式計畫書的功能。

三大匯出按鈕：

按鈕 功能說明 輸出格式與建議
🖨️ 預覽 / 列印 開新分頁，將所有填寫的資料（含圖片）排版進符合官方規格的計畫書表格中，並自動呼叫瀏覽器的列印對話框。 可直接選擇實體印表機列印，或在列印目標中選「另存為 PDF」保存。
⬇️ 下載 Word（.doc） 將排版好的計畫書網頁打包成一個 MicroSoft Word 格式的 .doc 檔案下載。 下載後，學生可以直接用 Word 開啟編輯，或貼入學校規定的 .odt 檔案範本中。
📋 複製全文純文字 將基本資料、學習動機、資源、方法、目標、週計畫列表、成效列表與家長評語，以清爽的排版整理成純文字，一鍵複製到剪貼簿。 適合需要將文字貼進學校其他線上申報系統（例如校務行政系統、自主學習平台）時使用。

💡 教師指導應用情境

為方便指導老師（如：自主學習指導老師、班導師、課程諮詢教師）在課堂中更有效率地應用此工具，以下提供三個推薦的指導情境：

情境一：自主學習第一堂課 — 構想引導與起草

  1. 引導全班開啟 AI輔助自主學習系統。
  2. 請學生先在「第 1 步：基本資料」填入主題構想，並點擊 AI 協助，讓 AI 給予題目範圍與聚焦度的建議。
  3. 學生修改好正式的「計畫名稱」後，進入「第 2 步：學習計畫」。請學生在動機、資源、方法、目標四個欄位各寫下一句最真實的想法。
  4. 點擊各欄的 AI 協助（外部模式會複製指令並打開 ChatGPT/Gemini，請學生貼上指令）。學生將 AI 產出的建議做個人化修改，並填回網頁。
  5. 要求學生點選右上方「儲存」，保存進度。

情境二：中期滾動式修正與追蹤

  1. 每隔數週的自主學習課，請學生在電腦上開啟網頁，在下拉選單選擇自己的名字並按「讀取」。
  2. 請學生到「第 3 步：預計進度」，針對過去幾週的進度，在「自我反思檢核」欄位下拉選擇完成狀況（A/B/C）。
  3. 點選「第 4 步：學習成效」，要求學生在成效表格中填寫當週心得，並「拍照上傳」目前的實體筆記、程式畫面或實驗成果，以作為學習歷程的真實佐證。
  4. 點擊「儲存」，確保成效紀錄與照片保存在這台電腦。

情境三：期末產出、成果發表與建檔

  1. 自主學習學期結束前，引導學生讀取暫存檔案。
  2. 到「第 4 步」點擊 重新計算統計，確認 A/B/C 的檢核統計次數已更新。
  3. 請學生邀請家長檢閱成效，在「家長給學生的回饋」欄位鍵入評語與家長姓名。
  4. 進入「第 5 步：匯出與列印」：
    • 若學校要求繳交電子檔，請學生點選「預覽 / 列印」並選擇「另存為 PDF」上傳至校內系統。
    • 若需要微調格式，點選「下載 Word」進行排版美化，這份檔案即可以直接作為多元表現的「自主學習計畫與成果」上傳至高中學習歷程檔案平台。
